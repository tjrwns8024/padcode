# 로드맵과 구현 상태

## 구현 완료

- 4×4 패드 그리드
- 패드 선택과 클릭 재생
- 키보드 트리거
- 한글/영어 DSL
- 7종 사운드 소스
- 9종 이펙트 체인
- CodeMirror 6 에디터
- DSL 자동완성
- DSL 실시간 린터
- `/help`, `/guide` 슬래시 명령어
- 프리셋 사이드바
- 사용자 MP3/WAV 등 오디오 샘플 업로드
- 업로드 샘플을 `샘플("이름")`으로 재생
- 하단 FFT 비주얼라이저
- RMS 기반 애니메이션
- WebM 녹음
- Gemini API 기반 AI 사운드 생성
- 우클릭 루프 표시 토글

## 현재 미구현 또는 부분 구현

- 셀별 미니 비주얼라이저
- 실제 반복 재생 스케줄러
- `스텝()` 패턴 기반 자동 시퀀싱
- 라이브/레코딩/플레이백 모드 전환 UI
- 패드 이벤트 타임라인 기록
- 이벤트 타임라인 기반 플레이백
- 그리드 크기 커스터마이즈
- 라이트/다크 테마 전환
- localStorage 또는 IndexedDB 기반 영속 저장
- WAV/MP3 인코딩 내보내기
- 별도 테스트 스위트

## 개발 단계

### 1단계: 현재 MVP 안정화

- 오디오 트리거 안정화
- DSL 린터와 자동완성 보강
- 샘플 업로드 UX 정리
- 녹음 결과 저장 흐름 개선
- AI 생성 DSL 검증 추가

### 2단계: 퍼포먼스 기능 확장

- 실제 루프 스케줄러 구현
- `스텝()` 패턴 기반 시퀀싱 구현
- 이벤트 타임라인 기록
- 플레이백 모드 구현
- BPM과 시퀀서 연동

### 3단계: 저장과 내보내기

- 셀 코드와 샘플 메타데이터 영속 저장
- 프로젝트 저장/불러오기
- WAV 인코딩 내보내기
- 녹음 파일 관리 UX 개선

### 4단계: 시각화 고도화

- 셀별 미니 비주얼라이저
- 악기 타입별 애니메이션
- 전체 믹스 비주얼 모드 다양화
- 라이트/다크 테마

### 5단계: 확장

- 그리드 크기 커스터마이즈
- 프리셋 시스템 확장
- 공유 기능
- 모바일 대응 검토
