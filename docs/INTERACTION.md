# 상호작용과 키보드 매핑

## 패드 그리드

PadCode의 기본 그리드는 4×4입니다. 각 셀은 하나의 독립 사운드 유닛이며, 셀마다 DSL 코드와 키 바인딩을 가집니다.

```text
1 2 3 4
Q W E R
A S D F
Z X C V
```

셀 ID는 `r{row}c{col}` 형식입니다.

| 행 | 키 | 셀 ID |
| --- | --- | --- |
| 1행 | `1 2 3 4` | `r0c0` ~ `r0c3` |
| 2행 | `Q W E R` | `r1c0` ~ `r1c3` |
| 3행 | `A S D F` | `r2c0` ~ `r2c3` |
| 4행 | `Z X C V` | `r3c0` ~ `r3c3` |

## 패드 조작

| 동작 | 결과 |
| --- | --- |
| 패드 클릭 | 해당 셀 선택 |
| 패드 안 재생 버튼 클릭 | 해당 셀의 DSL 사운드 재생 |
| 패드 우클릭 | 루프 표시 플래그 토글 |
| 패드 안 삭제 버튼 클릭 | 해당 셀 코드 삭제 |
| 키보드 입력 | 매핑된 셀의 사운드 재생 |

패드가 비어 있으면 재생하지 않습니다. 코드가 있는 패드는 코드 내용에 따라 색상과 라벨이 추론되어 표시됩니다.

## 키보드 입력 처리

전역 키보드 입력은 `hooks/useKeyboardTrigger.ts`에서 처리합니다.

- 물리 키 기준인 `KeyboardEvent.code`를 사용합니다.
- 한글 IME 상태와 무관하게 키 위치 기준으로 동작합니다.
- CodeMirror 에디터, 입력창, 텍스트 영역에 포커스가 있으면 패드 트리거를 막습니다.
- 재생 전 `ensureAudioContext()`를 호출해 브라우저 오디오 컨텍스트를 시작합니다.

## 에디터

오른쪽 `EditorPanel`은 선택된 셀의 DSL 코드를 편집합니다.

- 셀을 선택하면 해당 셀 코드가 표시됩니다.
- 코드 변경은 즉시 Zustand 셀 상태에 반영됩니다.
- 자동완성은 한글/영어 DSL 함수명을 모두 제공합니다.
- 린터는 현재 코드를 컴파일해 실패하면 에러 메시지를 표시합니다.

## 사이드바

왼쪽 `Sidebar`는 사용자 샘플과 프리셋을 관리합니다.

### 샘플 업로드

- 오디오 파일을 업로드하면 `샘플("이름")`으로 사용할 수 있습니다.
- 파일명은 DSL에서 쓰기 쉬운 이름으로 정규화됩니다.
- 같은 이름이 이미 있으면 `_2`, `_3`처럼 뒤에 번호를 붙입니다.
- 파일 크기는 20MB 이하만 허용됩니다.

### 프리셋 삽입

`SOUND BANK`의 프리셋을 클릭하면 선택된 패드에 DSL 코드가 삽입됩니다. 선택된 패드가 없으면 삽입 버튼은 비활성 상태입니다.

현재 기본 프리셋은 다음과 같습니다.

- `KICK`
- `SNARE`
- `HAT`
- `SINE`
- `BASS`
- `ARP`
- `CLAP`
- `DELAY`

## 비주얼라이저와 녹음

하단 `Visualizer`는 전체 믹스의 FFT와 RMS 값을 읽어 화면에 표시합니다.

오른쪽 녹음 패널에서 `REC`를 누르면 마스터 출력이 WebM으로 녹음됩니다. 다시 누르면 정지하고, 녹음 결과를 브라우저 오디오 컨트롤로 재생하거나 `.webm` 파일로 저장할 수 있습니다.

현재 녹음은 실제 오디오 출력만 저장합니다. 패드 이벤트 타임라인을 기록해 다시 시퀀싱하는 기능은 아직 구현되어 있지 않습니다.
