# DSL 레퍼런스

PadCode DSL은 한 패드의 사운드를 정의하는 체이닝 표현식입니다. 한글 함수와 영어 별칭을 함께 지원합니다.

## 기본 형태

```js
사인파(440).게인(0.5).딜레이(0.125)
```

또는 영어 별칭으로 작성할 수 있습니다.

```js
sin(440).gain(0.5).delay(0.125)
```

DSL은 반드시 하나의 표현식이어야 하며, 내부적으로 `build()`가 호출되어 `CompiledSound`로 변환됩니다.

## 사운드 소스

| 한글 | 영어 | 설명 |
| --- | --- | --- |
| `사인파(freq)` | `sin(freq)` | 사인파 오실레이터 |
| `노이즈()` | `noise()` | 화이트 노이즈 |
| `샘플("name")` | `sample("name")` | 내장 드럼 또는 사용자 업로드 샘플 |
| `플럭(freq)` | `pluck(freq)` | 기타/현악기처럼 튕기는 소리 |
| `베이스(freq)` | `bass(freq)` | 톱니파 기반 신스 베이스 |
| `피아노(freq)` | `piano(freq)` | 삼각파 기반 피아노풍 음색 |
| `오르간(freq)` | `organ(freq)` | AM 신스 기반 오르간 음색 |

## 내장 샘플 이름

`샘플("name")`은 먼저 사용자 업로드 샘플을 찾고, 없으면 내장 드럼 합성으로 폴백합니다.

| 이름 | 설명 |
| --- | --- |
| `kick` | 킥 드럼 |
| `snare` | 스네어 |
| `hat`, `hihat` | 하이햇 |
| `clap` | 클랩 |
| `tom` | 탐 |
| `cymbal` | 심벌 |

알 수 없는 샘플 이름이고 사용자 업로드 샘플도 없으면 짧은 노이즈 틱 소리로 폴백합니다.

## 이펙트 체인

| 한글 | 영어 | 값 | 설명 |
| --- | --- | --- | --- |
| `.게인(v)` | `.gain(v)` | `0~2` | 볼륨. 기본값은 `0.6` |
| `.로우패스(hz)` | `.lowpass(hz)` | `20~20000` | 저역 통과 필터 |
| `.밴드패스(hz)` | `.bandpass(hz)` | `20~20000` | 대역 통과 필터 |
| `.딜레이(time)` | `.delay(time)` | `0~1` | 딜레이 시간. 초 단위 |
| `.피치다운(semis)` | `.pitch(semis)` | 숫자 | 반음 단위 피치 다운 |
| `.스무딩(ms)` | `.smooth(ms)` | `0 이상` | 어택 스무딩 |
| `.에코(level)` | `.echo(level)` | `0~100` | 피드백 에코 |
| `.스텝(pattern)` | `.step(pattern)` | 문자열 | 스텝 패턴. `x`는 트리거, `.`은 쉼 |
| `.확률(p)` | `.prob(p)` | `0~1` | 트리거 확률 |

현재 `스텝`은 DSL 메타데이터로 저장되지만, 완성된 스케줄러처럼 패턴을 자동 반복 재생하는 구조는 아직 구현되어 있지 않습니다.

## 예제

### 킥 드럼

```js
샘플("kick").게인(1)
```

### 스네어

```js
샘플("snare").게인(0.8)
```

### 하이햇

```js
샘플("hat").게인(0.5)
```

### 딜레이 아르페지오

```js
사인파(660).게인(0.3).딜레이(0.125).에코(20)
```

### 신스 베이스

```js
베이스(80).게인(0.8).로우패스(300)
```

### 기타/현악기

```js
플럭(330).게인(0.6)
```

### 피아노풍 멜로디

```js
피아노(440).게인(0.5).딜레이(0.25)
```

### 확률적 노이즈 퍼커션

```js
노이즈().게인(0.2).밴드패스(3000).확률(0.6)
```

## 에디터 지원

CodeMirror 에디터는 다음 기능을 제공합니다.

- 한글/영어 DSL 함수 자동완성
- 함수 파라미터 힌트
- `compile()` 기반 실시간 린터
- `/help` 슬래시 명령어
- `/guide` 슬래시 명령어

## 컴파일 방식

DSL 컴파일은 `lib/dsl/builder.ts`의 `compile(code)`에서 처리합니다.

```ts
new Function(...argNames, `"use strict"; return (${code}).build();`);
```

즉, DSL은 전용 파서가 아니라 제한된 바인딩을 넘긴 JavaScript 표현식으로 평가됩니다. 이 구조는 구현이 간단하지만 완전한 보안 샌드박스는 아니므로, 신뢰할 수 없는 사용자 코드를 서버에서 실행하는 방식으로 확장하면 안 됩니다.
