# 기획 맥락

## 한 줄 정의

PadCode는 브라우저에서 동작하는 프로그래머블 런치패드입니다. 4×4 그리드의 각 셀에 한글/영어 DSL 코드를 입력해 사운드를 정의하고, 키보드로 실시간 트리거하며, 전체 믹스 비주얼라이저와 녹음을 제공합니다.

## 기획 배경

기존 음악 도구에는 서로 다른 장단점이 있습니다.

- 런치패드 도구는 즉시 연주하기 쉽지만, 보통 샘플 트리거 중심이라 표현 자유도가 낮습니다.
- 라이브 코딩 도구는 표현력이 높지만, 코드 중심이라 비전공자가 접근하기 어렵습니다.

PadCode는 이 둘을 결합합니다.

1. 코드로 음악을 정의하는 창작 경험
2. 키보드와 패드로 즉시 연주하는 퍼포먼스 경험
3. 녹음 후 다시 듣고 저장하는 루프 경험
4. 전체 믹스에 반응하는 시청각 경험

## 핵심 컨셉

### 셀은 독립 사운드 유닛

각 셀은 다음 요소를 함께 가집니다.

- DSL 코드
- 사운드 소스와 이펙트
- 키 바인딩
- 재생 상태
- 색상과 라벨

### 코드와 퍼포먼스의 분리

코드는 소리를 정의하고, 퍼포먼스는 그 소리를 언제 실행할지 결정합니다. 사용자는 먼저 각 패드에 사운드를 만들고, 이후 키보드나 클릭으로 연주할 수 있습니다.

### 시청각 통합

패드가 트리거되면 셀이 깜박이고, 전체 믹스 비주얼라이저가 FFT와 RMS 값에 반응합니다. 현재 구현은 하단 전체 비주얼라이저 중심이며, 셀별 미니 비주얼라이저는 계획 항목입니다.

## 사용자 시나리오

### 셀 기반 사운드 제작

1. 패드를 선택합니다.
2. 오른쪽 에디터에 DSL 코드를 입력합니다.
3. 패드 클릭 또는 키보드로 사운드를 테스트합니다.
4. 프리셋이나 AI 생성 기능으로 코드를 빠르게 채울 수 있습니다.

### 라이브 퍼포먼스

1. 여러 패드에 킥, 스네어, 하이햇, 베이스, 멜로디를 배치합니다.
2. `1234 / QWER / ASDF / ZXCV` 키로 패드를 트리거합니다.
3. 하단 비주얼라이저로 전체 믹스 반응을 확인합니다.

### 녹음과 저장

1. 하단 녹음 패널에서 `REC`를 누릅니다.
2. 패드를 연주합니다.
3. 다시 버튼을 눌러 녹음을 멈춥니다.
4. 결과를 브라우저에서 재생하거나 WebM 파일로 저장합니다.

## 차별점

- 셀마다 독립 코드 유닛을 둡니다.
- 단순 샘플 버튼이 아니라 DSL 기반 사운드 로직을 지원합니다.
- 한글과 영어 DSL을 동시에 지원합니다.
- 별도 장비 없이 브라우저만으로 런치패드형 연주 경험을 제공합니다.
- AI 생성 기능으로 자연어에서 DSL 코드로 진입할 수 있습니다.

## 결정 기록

| 초기 가정 | 검토 결과 | 최종 결정 |
| --- | --- | --- |
| Strudel 임베드 사용 | 한글 DSL과 코드 통제력이 부족함 | Tone.js + 자체 DSL 래퍼 |
| 녹음/재생은 후순위 | 발표 데모에 필요함 | 녹음 기능을 MVP에 포함 |
| 한글 DSL만 지원 | 영어 사용자와 개발 편의도 필요함 | 한글/영어 DSL 동시 지원 |
| JSON 내보내기 중심 | 실제 음악 결과물이 필요함 | 오디오 녹음 파일 저장 지원 |
