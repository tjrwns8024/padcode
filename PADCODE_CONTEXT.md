# PadCode — 프로젝트 맥락 정리

> 라이브 코딩 + 런치패드 + 루퍼를 결합한 브라우저 기반 인터랙티브 음악 도구

## 1. 한 줄 정의

브라우저에서 동작하는 **프로그래머블 런치패드**. 4x4 그리드의 각 셀에 한글/영어 DSL 코드를 입력하여 사운드를 정의하고, 키보드로 실시간 트리거하며, 셀 위 비주얼라이저가 반응한다. 연주를 녹음/재생/내보내기할 수 있다.

---

## 2. 기획 의도

기존 도구의 한계를 결합으로 해결한다.

- **런치패드 도구**: 샘플 트리거에 머물러 표현 자유도가 낮음
- **라이브 코딩 도구** (Strudel, Sonic Pi 등): 코드 중심이라 비전공자 진입 장벽이 큼

PadCode는 다음을 동시에 제공한다.

1. 코드로 음악을 정의하는 **창작 경험**
2. 키보드/패드로 즉시 연주하는 **퍼포먼스 경험**
3. 녹음 후 반복 재생하는 **루프 경험**
4. 악기별 비주얼이 반응하는 **시청각 경험**

---

## 3. 핵심 컨셉

### 3-1. 셀 = 악기 + 코드 + 비주얼

각 런치패드 셀은 하나의 독립 사운드 유닛이다.

- **사운드**: 악기/샘플/패턴/이펙트
- **로직**: 반복, 조건, 확률, 루프
- **비주얼**: 악기 성격에 따른 시각화
- **인터랙션**: 키보드/클릭 트리거

### 3-2. 코드와 퍼포먼스의 분리·결합

- 코드 = 소리 정의
- 퍼포먼스 = 실행 타이밍

두 축을 분리하면서도 한 인터페이스에 결합한다.

### 3-3. 시청각 통합 UX

셀을 누르면 소리와 함께 셀 색상, 미니 비주얼, 전체 믹스 비주얼이 모두 반응한다.

---

## 4. 기술 스택 (확정)

| 영역 | 기술 |
| --- | --- |
| 프레임워크 | **Next.js** (프론트엔드 SPA, 백엔드 없음) |
| 오디오 엔진 | **Tone.js** (Web Audio API 위 고수준 라이브러리) |
| DSL | 한글 + 영어 DSL 동시 지원, Tone.js 변환 레이어 직접 구현 |
| 비주얼라이저 | Canvas / WebGL, Tone.js FFT/Waveform analyser 활용 |
| 내보내기 | Tone.js Recorder 또는 MediaRecorder API + WAV 인코딩 |
| 디자인 톤 | 터미널 / 코드 에디터 스타일 (claude-agent-viz 레퍼런스) |
| 상태관리 | React state + Zustand 또는 Redux Toolkit (검토 단계) |
| 저장 | localStorage 기반 JSON (백엔드/DB 없음) |

> ⚠️ Strudel 임베드는 **검토 후 폐기**. 한글 DSL 자유도와 코드 통제력을 위해 Tone.js로 결정.

## 6. DSL 명세

### 6-1. 한글 DSL 함수 (12개+ 전부 MVP 필수)

| 분류 | 함수 |
| --- | --- |
| 음원 | 사인파, 샘플, 노이즈 |
| 게인/필터 | 게인, 로우패스, 밴드패스 |
| 이펙트 | 딜레이, 피치다운, 스무딩, 에코 |
| 패턴 | 스텝, 확률 |
| 재생 모드 | 반복 (= loop) |

### 6-2. 영어 DSL (한글과 동일 동작)

`sin`, `sample`, `noise`, `gain`, `lowpass`, `bandpass`, `delay`, `pitch`, `smooth`, `echo`, `step`, `prob`, `loop`

### 6-3. 코드 예시

```
사인파(10000)
  .피치다운(30)
  .스무딩(30)
  .에코(10)
```

```
sample("kick")
  .step("x..x..x.")
```

```
사인파(440)
  .게인(0.5)
  .로우패스(1200)
  .딜레이(1/8)
```

## 7. 입력 시스템

### 7-1. 키보드 매핑

QWER / ASDF / ZXCV → 4x4 그리드 셀에 대응

### 7-2. 모드 (3종, 전부 MVP 필수)

| 모드 | 동작 |
| --- | --- |
| **라이브** | 키보드 입력 즉시 재생, 코드 변경 즉시 반영 |
| **레코딩** | 트리거 시간 기록 → 시퀀스 생성 |
| **플레이백** | 기록된 연주 타임라인 재생 |

### 7-3. 재생 모드

- **one-shot**: 키 누르면 한 번 재생
- **loop**: 키보드 토글 (on/off), `.반복()` 또는 `.loop()` DSL로도 설정 가능
- DSL과 UI 우클릭 메뉴 양쪽에서 설정 가능

## 13. 사용자 시나리오

### 시나리오 1 — 셀 기반 사운드 제작

1. 셀을 선택한다.
2. 우측 코드 에디터에 한글/영어 DSL 코드를 입력한다.
3. 셀 위에 악기 타입별 비주얼이 적용된다.
4. 클릭 또는 키보드로 셀을 테스트 재생한다.

### 시나리오 2 — 라이브 퍼포먼스 녹음

1. 레코딩 모드를 활성화한다.
2. 키보드로 여러 셀을 순차 트리거한다.
3. 트리거 시점이 기록된다.
4. 녹음 종료 후 재생, 전체 믹스 비주얼로 결과를 확인한다.

### 시나리오 3 — 루프 퍼포먼스 제작

1. 드럼/베이스/코드/멜로디 셀을 각각 세팅한다.
2. 각 셀에 loop / one-shot 속성을 설정한다.
3. 키보드로 셀을 조합해 실시간 퍼포먼스를 만든다.
4. 결과를 저장하고 다시 재생한다.

### 시나리오 4 — 완성 데모 (MVP "이거 완성이야" 기준)

**라이브 연주 데모**: 키보드로 셀들을 즉시 트리거하며 음악 만드는 모습을 시연.

## 14. 차별점

1. **셀마다 독립 코드 유닛** — 단순 샘플 버튼이 아닌 음악 로직 유닛
2. **코드와 연주의 분리·결합** — 코드 작성과 퍼포먼스 기록이 별개
3. **시청각 통합 UI** — 악기별 비주얼라이저 반응
4. **런치패드의 웹화** — 전용 장비 없이 브라우저만으로
5. **확장 가능 DSL** — 한글/영어 동시 지원
